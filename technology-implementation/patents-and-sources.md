# Patents And Sources

## Core PANN patents

### US9390373B2

Title: Neural network and method of neural network training

Source: https://patents.google.com/patent/US9390373B2/en

Key implementation relevance:

- Describes a progressive neural network with inputs, synapses, distributors,
  multiple corrective weights per synapse, neurons, and a weight correction
  calculator.
- Input values are mapped into intervals.
- Corrective weights can be indexed by input index `i`, interval index `d`, and
  neuron index `n`, giving the natural implementation tensor `W[i, d, n]`.
- A distributor selects one or more corrective weights in correlation with the
  input value.
- A neuron sum is generated by adding selected corrective weights, optionally
  weighted by coefficients of impact.
- Training modifies active corrective weights using the deviation between the
  neuron sum and the desired output.
- Two update styles are disclosed at a high level:
  - difference-based correction
  - ratio/deviation-coefficient correction

Google Patents lists the patent as active and gives an anticipated expiration
date of 2035-03-06.

### US9619749B2

Title: Neural network and method of neural network training

Source: https://patents.google.com/patent/US9619749B2/en

Key implementation relevance:

- Extends the architecture into memory elements and physical/analog/digital
  realizations.
- Describes corrective weights retained by memory elements.
- Mentions electrical and magnetic characteristics as possible physical weight
  representations.
- Mentions resistors, memristors, transistors, capacitors, photoresistors, and
  magnetic dependent resistors as possible memory/weight elements.
- Describes supplementary training and adding/removing corrective weights,
  inputs, synapses, or neurons.
- Supports the idea that a software prototype can model the same structure in
  ordinary memory.

### US10423694B2

Title: Neural network and method of neural network training

Source: https://patents.google.com/patent/US10423694B2/en

Key implementation relevance:

- Describes corrective weights organized in arrays.
- Training images may be received as arrays or codified as arrays.
- Neurons generate a neuron-sum array by summing selected corrective weights.
- A controller receives desired images in array form, determines a deviation
  array, and modifies a corrective-weight array.
- This is the most relevant source for a matrix/batch implementation.

### US11494653B2

Title: Neural network and method of neural network training

Source: https://patents.google.com/patent/US11494653B2/en

Key implementation relevance:

- Describes a controller that modifies corrective weights from output deviation.
- Adds a structure-forming module that rearranges connections during training.
- Adds a signal allocation module that distributes input signals among network
  elements during training.
- Adds a training module coordinating structure formation, signal allocation,
  and correction to control training in real time.
- Useful for later adaptive-structure experiments, not required for a minimal
  prototype.

## PANN publications

### Analog and Digital Modeling of a Scalable Neural Network

Source: https://progress.ai/analog-digital-modeling-scalable-neural-network/

Implementation relevance:

- Describes the p-network/p-neuron model.
- Explains analog and digital versions.
- Gives the matrix intuition: input images can be represented as binary
  activation arrays, selected corrective weights are summed, and training
  corrections are applied to selected weights.
- Reports tests against IBM SPSS Statistics 22 and NeuroSolutions.

### Fast trained neural network

Source: https://progress.ai/fast-trained-neural-network/

Implementation relevance:

- Gives a readable training algorithm.
- Defines a p-neuron with a distributor and multiple corrective weights.
- Describes a difference update:
  `delta_n = (target_n - sum_n) / S`, where `S` is the number of active
  synapses/weights contributing to neuron `n`.
- Describes a ratio update:
  `factor_n = target_n / sum_n`, with active weights multiplied by the factor.
- Reports a simple Python implementation and IRIS benchmark.

## PANC sources

### PANC technology page

Source: https://progress.ai/panc-ai-technology/

Implementation relevance:

- Describes PANC as a comparator-centric architecture.
- Recognition is framed as structured comparison against libraries of examples
  or analogues.
- Decisions are derived from explicit comparative matches.
- Public information is enough to sketch a nearest-neighbor/associative
  comparator prototype, but not enough to reproduce a proprietary PANC system.

### Springer chapter: Comparative Recognition Technology for Artificial Intelligence

Source: https://link.springer.com/chapter/10.1007/978-3-032-13056-3_17

Implementation relevance:

- Public abstract and metadata support the existence of a peer-reviewed PANC
  publication.
